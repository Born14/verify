# Verify Calibrated Change Receipt -- annotated sample (v1.3 schema)

A complete sample receipt from the v1.3 demo fixture, with short annotations on each block. The receipt is what a reviewer sees on every pull request; this page exists so the v1.3 format is legible before install.

The v1.3 schema adds two arrays to the receipt: `suppressions` (findings deliberately silenced via in-band `# verify:ignore` comments with a non-empty reason) and `suppression_warnings` (typo'd or unmatched suppression comments). Both are part of the packet digest. v1.3 also adds the first calibrated Terraform shape (`TF-SG-WORLD-OPEN-INGRESS-01`, warning-only).

For the historical v1 sample, see [VERIFY-RECEIPT-SAMPLE.md](VERIFY-RECEIPT-SAMPLE.md).

Inputs that produced this receipt:

- repo: `verify-demo/change-receipt-fixture`
- pull request: `#1`
- source commit: `1111111111111111111111111111111111111111`
- generated_at: `2026-05-06T00:00:00.000Z`
- Action bundle: pinned (the bundle version contributes to the digest)

Identical inputs produce a byte-identical receipt. Re-running the same workflow against the same commit will produce the same digest.

The v1.3 canonical demo digest is `sha256:0d7d9d941a672e37f2a31e0d8db8b20ce7ca416d57cf989c89deac7ca958b0e4`. This document's sample uses placeholder content for illustration; the canonical demo fixture is generated by the engine and the digest above is the audit anchor.

What the reviewer sees:

- **schema** -- the receipt format version. Stable contract.
- **scope** -- per-surface file counts actually inspected.
- **checks calibrated** -- count of checks that ran. Each has a published precision number.
- **result** -- one-line summary. `CLEAR` or `N FINDINGS`.
- **digest** -- SHA-256 over the receipt content (including the new `suppressions` and `suppression_warnings` arrays). Identical inputs produce identical digests.

## Block 6 -- NOT CHECKED

```
This receipt does NOT attest to the items below. They are outside calibrated coverage at v1.3.

- [NOT CHECKED] surface:terraform -- Most Terraform .tf surface area is not parsed; only the security-group ingress shape (TF-SG-WORLD-OPEN-INGRESS-01, warning-only) is calibrated under the v1.1 same-module resolver.
- [NOT CHECKED] surface:cloudformation -- CloudFormation YAML/JSON templates are not parsed.
- [NOT CHECKED] surface:helm-templated -- Helm chart templates ({{ ... }} expressions) are skipped at detector level. Only files parseable as raw YAML are inspected.
- [NOT CHECKED] surface:kustomize-overlays -- Kustomize strategic-merge patches and overlay resolution are not modeled.
- [NOT CHECKED] state:runtime-cloud -- Live cloud-account state (deployed IAM policies, running pods, attached security groups) is not consulted.
- [NOT CHECKED] semantics:business-logic -- Application correctness, business-rule integrity, and user-facing behavior are not checked.
- [NOT CHECKED] semantics:intent -- Author intent (by-design vs. mistake) is not inferred.
- [NOT CHECKED] completeness:recall -- This receipt does not establish what verify missed. Calibration measures precision on pinned corpora, not recall on the inspected changeset.
- [NOT CHECKED] completeness:uncalibrated-shapes -- Shipped-but-uncalibrated detectors are excluded from change receipts until calibrated.
```

What the reviewer sees:

- The exact boundary of what the receipt attests to.
- The Terraform surface entry is narrowed at v1.3 to acknowledge the one calibrated shape; everything else under Terraform remains uncovered.
